# 画廊元数据工作流 Prompt

- 每次执行前先读取这个文件。
- 不要请求体过大。
- 先读取当前激活存储路径，再开始处理图片。
- 每次只处理一个国家。
- 单批最多 5 张。
- 不要把整库元数据、超长日志、大拼图一次性塞进请求。
- 不要调用 groq 和 gemini。
- 需要 AI 的步骤，只使用当前对话里的这个模型。
- 上传链路只负责保存图片并标记待处理，不在上传请求里直接识别。
- 后续由小批次导出任务把图片路径和当前元数据交给当前对话模型识别，再回写 `city/place/subject/scene_summary`，并按 `place -> city` 规则生成标题。
- 如果用户手动改标题，标题优先级最高：先把 `place = 标题`，再根据 `place` 修正 `city`。
- 国家介绍也由当前对话模型生成后再回写，不在脚本里直接调用第三方模型。
