# iCloud 本地存储公网分享设计

**目标**

在继续使用本机 iCloud Drive 目录存图的前提下，让这个相册项目同时具备两种运行形态：

- 本地后台：只给站点管理员本人使用，负责上传、分类、修改国家说明
- 公网展示：只开放公共展示页，供其他人通过浏览器访问

**现状**

- 项目当前默认使用 `PHOTO_STORAGE=icloud`
- 已支持通过 `PUBLIC_SITE_ONLY=true` 关闭后台与 API
- 当前应用默认 `app.run(... host="0.0.0.0")`，不利于把后台默认限制在本机
- 仓库内缺少本地双实例启动脚本、停止脚本、状态检查脚本，以及面向 Cloudflare Tunnel 的配置模板

**推荐方案**

采用“本机双实例 + Cloudflare Tunnel”的结构：

- 后台实例：`127.0.0.1:5001`
- 展示实例：`127.0.0.1:5002`
- 两个实例都继续读取同一个 iCloud 目录
- Cloudflare Tunnel 仅转发到 `127.0.0.1:5002`

这样可以做到：

- 后台不暴露到公网
- 展示页地址可以公开分享
- 不需要迁移现有图片和元数据
- 不需要家用路由器端口映射

**功能边界**

- 应用新增运行地址配置，默认改为仅监听 `127.0.0.1`
- 保留端口配置能力
- 公网展示实例继续只读展示，不生成后台入口
- 公网展示实例不应在页面请求中补写国家描述，避免只读实例隐式写入本地元数据
- 提供一套本地脚本：
  - 启动后台与展示实例
  - 停止后台与展示实例
  - 查看运行状态和日志位置
  - 发起 Cloudflare Quick Tunnel 临时分享
- 提供固定 Tunnel 的配置样板和中文操作文档

**安全要求**

- 后台默认绑定 `127.0.0.1`
- 展示实例启用 `PUBLIC_SITE_ONLY=true`
- 对外只发布展示端口，不发布后台端口
- 运行模式默认关闭 Flask debug

- 修改 [app.py](/Users/mark/vscode/vscode1/app.py)
- 更新 [.env.example](/Users/mark/vscode/vscode1/.env.example) 与 [.env](/Users/mark/vscode/vscode1/.env)
- 新增测试目录与回归测试
- 新增本地运行/分享脚本
- 新增 Cloudflare Tunnel 配置样板
- 新增中文使用文档

**验收标准**

- 执行启动脚本后，后台实例可在本机 `5001` 正常访问
- 执行启动脚本后，展示实例可在本机 `5002` 正常访问
- 展示实例的 `/admin`、`/admin/login`、`/api/*` 返回 `404`
- 展示实例公共页面不触发国家描述补写
- Quick Tunnel 脚本能够把 `5002` 分享为临时公网地址
- 文档清楚区分“已自动完成”和“用户仍需手动完成”的步骤
